## 2. 需求与决策（本次确定）

### 2.1 页面结构

- 仅保留 2 个 Tab：
  - `基本设置`：个人资料表单 +（从“安全设置”挪入的）账户安全信息展示
  - `修改密码`：旧密码/新密码/确认密码
- 原“安全设置”页不再保留；其内容以“展示信息 + 操作入口”的方式并入“基本设置”。
- 角色编辑策略：个人中心中的 `角色` 始终只读；仅管理员在“用户管理”中修改用户角色。

### 2.2 修改密码校验规则（不加复杂度/格式限制）

- `oldPassword` 必填
- `newPassword` 必填，且 **`newPassword !== oldPassword`**
- `confirmPassword` 必填，且 **`confirmPassword === newPassword`**

## 3. UI/交互设计（建议实现形态）

### 3.1 基本设置（合并安全信息）

在“基本设置”中把“手机”与“姓名/用户名/角色”保持同一表单样式（只读输入框）：

- 字段名：`手机`
- 展示：脱敏手机号（`149****2300`）；无手机号时显示“未绑定手机号”
- 说明：本期仅展示，不做绑定/换绑/查看完整号等流程

> 备注：原 `@vben/common-ui` 的 `ProfileSecuritySetting` 使用 `Switch` 并触发 `change` 事件，但当前项目无对应后端能力；本期不再使用 `Switch` 承载该语义，避免“装饰开关”误导。

### 3.2 修改密码

- 提交成功后提示“修改成功”
- 修改密码成功后会强制退出：前端提示后自动退出并跳转登录页

## 4. 接口设计与契约（需要补齐到后端与 API_CONTRACT）

### 4.1 更新个人资料

建议新增：

- `PUT /api/user/profile`

请求体（JSON）：

```ts
type UpdateMyProfileBody = {
  realName: string;        // 展示名/姓名
  introduction?: string;   // 个人简介
  avatar?: string;         // 头像 URL（可选）
};
```

响应：通用结构（`code===0`）返回更新后的 `UserInfo`（建议直接复用 `GET /api/user/info` 的返回结构，便于前端刷新 store）。

约束建议：

- `realName` 不能为空（400）
- `introduction` 可为空；建议限制长度（例如 0~200）但不强制（本期可先不加）
- `avatar` 可为空；若为上传接口返回的 `/api/uploads/*` 则可直接存储

### 4.2 修改密码

建议新增：

- `PUT /api/user/password`

请求体（JSON）：

```ts
type ChangeMyPasswordBody = {
  oldPassword: string;
  newPassword: string;
};
```

响应：通用结构（`code===0`），`data` 可为空对象 `{}` 或返回最新 `UserInfo`（二者均可，前端主要需要成功/失败信号）。

错误与约束：

- `oldPassword`/`newPassword` 不能为空（400）
- `newPassword === oldPassword`（409 或 400，建议 409：冲突）
- `oldPassword` 不正确（403）

会话策略（已确认）：

- 修改密码成功后**删除该用户所有 session**，强制退出重新登录

## 6. 后端开发拆分（待实施）

### 6.1 数据结构

并在 `GET /api/user/info` 中映射：

- `introduction` 取 `introduction`（无则空串）

### 6.2 路由实现

建议在 `backend/src/routes/user.ts` 内新增：

- `PUT /user/profile`
- `PUT /user/password`

核心点：

- `PUT /user/profile`：仅允许更新当前登录用户（从 `ctx.state.currentUser` 获取），不要支持传 userId
- `PUT /user/password`：
  - 校验旧密码：复用 `verifyPassword`
  - 写入新 hash/salt：复用 `hashPassword`
  - 按会话策略处理：若选 A，需要新增“按 userId 删除 sessions”的 DB 方法

### 6.3 契约更新
