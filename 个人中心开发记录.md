## 2026-03-07 v1.0

### 目标
- 将个人中心从纯展示页补齐为可用功能
- 合并原“安全设置”内容到“基本设置”
- 最终仅保留两个 Tab：`基本设置`、`修改密码`

### 细节优化记录
- 基本设置 UI：移除“账户安全/账户密码”独立块
- 手机并入表单，用只读输入框脱敏展示
- `username` 作为账号标识保持只读，避免影响历史数据一致性

### 现状梳理
- 路由：`/profile`
- 页面入口：`vue-vben-admin/apps/web-ele/src/views/_core/profile/index.vue`
- 基本设置：`base-setting.vue`
- 修改密码：`password-setting.vue`
- 历史 `security-setting.vue` 文件仍在，但不再作为独立 Tab 引用

### 已有接口
- `GET /api/user/info`
- `PUT /api/user/profile`
- `PUT /api/user/password`

### 页面结构决策
- `基本设置`：资料表单 + 账户安全展示信息
- `修改密码`：旧密码、新密码、确认密码
- 角色始终只读，由管理员在“用户管理”中修改

### 修改密码规则
- `oldPassword` 必填
- `newPassword` 必填，且不能与 `oldPassword` 相同
- `confirmPassword` 必填，且必须与 `newPassword` 相同

### UI/交互
- 手机脱敏展示；无手机号时显示“未绑定手机号”
- 修改密码成功后提示成功，并强制退出到登录页

### 接口设计
#### `PUT /api/user/profile`
- 入参：
  - `realName`
  - `introduction?`
  - `avatar?`
- 约束：
  - `realName` 不能为空
  - `introduction` 可为空
  - `avatar` 可为空

#### `PUT /api/user/password`
- 入参：
  - `oldPassword`
  - `newPassword`
- 约束：
  - 新旧密码不能相同
  - 旧密码错误返回失败
- 会话策略：
  - 成功后删除该用户所有 session，强制重新登录

### 前端改动点
- `index.vue`：仅保留两个 Tab
- `base-setting.vue`：接入资料保存
- `password-setting.vue`：接入修改密码与前端校验
- `src/api/core/user.ts`：新增个人中心相关 API

### 后端改动点
- `UserDoc` 增加：
  - `real_name?`
  - `introduction?`
- `GET /api/user/info` 增加对应字段映射
- 在 `backend/src/routes/user.ts` 新增资料更新、修改密码接口
- 在 `backend/API_CONTRACT.md` 补齐契约说明

### 验收清单
- 页面可正确回填 `realName/username/roles/introduction`
- 更新资料后能保存并刷新用户信息

- 三项密码校验生效
- 旧密码错误能正确提示
