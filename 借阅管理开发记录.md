## 2026-03-16 v1.3

### 范围
- 页面：`vue-vben-admin/apps/web-ele/src/views/library/borrows/index.vue`
- 前端接口：`vue-vben-admin/apps/web-ele/src/api/library/borrows.ts`
- 后端接口：`backend/src/routes/borrows.ts`
- 接口契约：`backend/API_CONTRACT.md`

### 问题说明
- 当前管理员“借阅管理”页未提供 `reserved`（待取书）记录的正常流转入口。
- 当前页面将 `reserved` 也纳入“还书”操作范围，存在“待取书直接还书”的错误流转风险。
- 当前模型将“预约时间/借出时间”“待取截止/应还时间”复用为同一组字段，语义不清。
- 当前模型将“待取超期”和“借阅逾期”折叠为同一个 `overdue`，不利于展示、操作与后续规则扩展。

### 字段模型调整
- 借阅记录时间字段拆分为：
  - `reserved_at`：预约时间
  - `pickup_due_at`：待取截止时间
  - `borrowed_at`：实际借出时间
  - `return_due_at`：应还时间
  - `returned_at`：实际归还时间
- 保留 `borrow_days`、`fine_amount` 等现有业务字段。
- 旧字段迁移原则：
  - 原 `borrow_date` 不再继续承担双重语义
  - 原 `due_date` 不再继续承担双重语义
  - 原 `return_date` 收敛到 `returned_at`

### 状态定义
- 借阅记录状态枚举调整为：
  - `reserved`：待取书
  - `reserve_overdue`：待取超期
  - `borrowed`：借阅中
  - `borrow_overdue`：借阅逾期
  - `returned`：已归还
  - `canceled`：已取消
- 不再依赖 `raw_status + status` 组合推导业务状态。
- 本次仅调整借阅状态流转，不引入信用分、罚款自动联动。

### 页面改动点
- 借阅管理操作区按状态细分：
  - 待取书：显示“确认借出”“取消预约”
  - 待取超期：仅显示“取消预约”
  - 借阅中/借阅逾期：显示“还书”
  - 已归还：保留“详情”
- 表格时间列拆分并正名：
  - `预约时间`
  - `待取截止时间`
  - `借出时间`
  - `应还时间`
  - `实际归还时间`
- 各操作抽屉补充描述信息，覆盖记录基础信息与时间信息，减少对“详情”按钮的依赖。
- 状态文案区分：
  - `待取书`
  - `待取超期`
  - `借阅中`
  - `借阅逾期`
  - `已归还`
  - `已取消`

### 接口改动点
- 管理员新增“取消预约”能力，用于处理 `reserved`、`reserve_overdue` 记录。
- 管理员“确认借出”复用现有 `POST /api/borrows/borrow`：
  - 若存在同用户同 ISBN 的待取书记录，则转为 `borrowed`
  - 若记录已进入 `reserve_overdue`，则禁止确认借出
- `PUT /api/borrows/:recordId/return` 补充限制：
  - `reserved` / `reserve_overdue` 记录不可直接还书
- `GET /api/borrows` / `GET /api/borrows/my` 返回结构同步切换到新字段与新状态枚举。
- 需要同步更新 `backend/API_CONTRACT.md`，明确新状态与新时间字段含义。

### 数据迁移约定
- 对已有 `status='reserved'` 且未超过原 `due_date` 的记录：
  - 迁移为 `status='reserved'`
  - `reserved_at = borrow_date`
  - `pickup_due_at = due_date`
- 对已有 `status='reserved'` 且已超过原 `due_date` 的记录：
  - 迁移为 `status='reserve_overdue'`
  - `reserved_at = borrow_date`
  - `pickup_due_at = due_date`
- 对已有借阅中/逾期未归还记录：
  - `borrowed_at = borrow_date`
  - `return_due_at = due_date`
  - 若当前已逾期，则迁移为 `borrow_overdue`
- 对已有已归还记录：
  - `borrowed_at = borrow_date`
  - `return_due_at = due_date`
  - `returned_at = return_date`
  - `status='returned'`
- 对已有已取消记录：
  - 根据是否存在预约阶段数据，尽量保留 `reserved_at` / `pickup_due_at`
  - `status='canceled'`

### 业务约束
- 用户预约成功后先进入“待取书”，并已占用库存。
- 线下核验通过后，管理员执行“确认借出”，记录转为“借阅中”，不重复扣库存。
- 待取书超期后，记录状态变为“待取超期”，不允许再确认借出，仅允许取消预约并释放库存。
- 管理员取消预约后，需要回补库存，且不超过 `total_stock`。
- 借阅逾期与待取超期为两种独立状态，后续可分别接入信用分/罚款规则。
- 本次不处理信用分扣减、不处理罚款自动计算。

### 验收点
- 管理员列表中的待取书记录显示“确认借出”“取消预约”两个入口。
- 待取超期记录不显示“确认借出”，仅可“取消预约”。
- 待取书记录无法再通过“还书”入口完成状态变更。
- 管理员确认借出待取书后，记录转为“借阅中”，库存不重复扣减。
- 管理员取消待取书/待取超期后，记录转为“已取消”，库存正确回补。
- 借阅中、借阅逾期记录仍可正常还书。
- 历史记录迁移后，表格可正确展示预约时间、待取截止时间、借出时间、应还时间、实际归还时间。
- 接口返回中，“待取超期”和“借阅逾期”可直接通过状态枚举区分。
- 已归还记录保留“详情”查看能力，其它状态主要通过操作抽屉查看记录信息。

### 实现结果
- 后端借阅记录模型已拆分时间字段：
  - `reserved_at`
  - `pickup_due_at`
  - `borrowed_at`
  - `return_due_at`
  - `returned_at`
- 后端新增借阅记录迁移与状态刷新逻辑：
  - 启动时自动迁移旧借阅记录到新字段结构
  - 启动与接口访问时自动将超期记录刷新为：
    - `reserve_overdue`
    - `borrow_overdue`
- 后端借阅流转已调整：
  - 用户预约生成 `reserved`
  - 管理员确认借出将 `reserved -> borrowed`
  - 待取超期仅允许取消预约，不允许确认借出
  - `reserved / reserve_overdue` 不允许直接还书
  - 取消预约接口支持：
    - 用户取消自己的预约
    - 管理员取消任意待取/待取超期记录
- 前端借阅管理页已调整：
  - 表格拆分展示预约时间、待取截止时间、借出时间、应还时间、实际归还时间
  - 操作按钮按新状态显示：
    - `reserved`：确认借出、取消预约
    - `reserve_overdue`：取消预约
    - `borrowed / borrow_overdue`：还书
    - `returned`：详情
  - 各写操作统一走抽屉 + 二次确认
- 用户借阅记录页已同步新状态与取消预约判断，避免与后端新状态脱节。
