# 启动脚本优化计划 v1.0（2026-03-14）

## 背景
- `start-dev.ps1` 目前含 `#Requires -Version 7.0`，在资源管理器右键“使用 PowerShell 运行”时通常会用 Windows PowerShell 5.1 打开，导致脚本直接拒绝执行。
- `codex` Tab 里已通过 `wsl.exe --cd <wslRoot>` 进入项目目录，但又额外使用 `codex -C <wslRoot>`，存在重复定位。

## 目标
- （本次已确认范围）仅去掉 WSL/Codex 的重复定位，保持行为不变、参数更直观。

## 范围
- 仅涉及启动脚本：`start-dev.ps1`（必要时可新增一个可选的启动入口文件，例如 `start-dev.cmd`，用于强制使用 `pwsh` 启动）。
- 不改业务代码、不改后端/前端逻辑。

## 方案与改动点
### A. 去掉 WSL/Codex 重复定位（本次执行）
- `codex` Tab：保留 `wsl.exe --cd <wslRoot>`，将 `exec codex -C '<wslRoot>'` 简化为 `exec codex`（避免重复定位）。

### B. 额外提供“使用 PowerShell 7 运行”（可选，需要改注册表）
- 提供一个 `.reg`（或手动步骤）在右键菜单中新增 “使用 PowerShell 7 运行”，指向 `pwsh.exe -File "%1"`。
- 风险：涉及系统/用户注册表，可能被安全软件拦截；不同 Windows 版本/策略可能有差异。

## 验收点
- `codex` Tab 进入项目目录后直接启动 `codex`，不再使用 `-C` 传参。

## 变更记录
- 2026-03-14：已修改 `start-dev.ps1`，移除 `codex -C '<wslRoot>'` 的重复定位参数。
- 2026-03-14：已修改 `start-dev.ps1`，将 Windows Terminal tab 标题从 `codex` 调整为 `wsl: codex`。
