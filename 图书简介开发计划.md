# 图书简介开发计划 v1.0（2026-03-07）

## 1. 范围
### 1.1 目标
- 在“图书管理”新增字段：`introduction`（图书简介）。
- 主页列表表格不展示简介；仅在“新增图书/编辑图书”的抽屉表单中展示与编辑。
- 批量导入（Excel）同步支持简介字段（可选列）。

### 1.2 不做
- 不在列表筛选条件中增加简介相关搜索（避免引入低价值复杂度）。
- 不在借阅流程的“图书查找/选择”接口中使用简介（借阅只需要基本信息即可）。

## 2. 数据与校验约束
- `introduction`：可选，字符串，去除首尾空格。
- 长度限制：`<= 300` 字符（后端强校验；前端输入框展示字数限制）。

## 3. 接口/后端改动点（backend）
### 3.1 数据模型
- `BookDoc` 增加字段：`introduction?: string`

### 3.2 业务接口
- `GET /api/books`：列表项返回增加 `introduction`（前端表格不展示，但编辑抽屉需要回填）。
- `POST /api/books`：支持写入 `introduction`。
- `PUT /api/books/:isbn`：支持更新 `introduction`。

### 3.3 批量导入
后端解析 Excel 时新增可选列：
- 表头支持：`introduction` 或 `简介`

导入策略（与现有冲突策略对齐）：
- 新书创建：写入 `introduction`（如为空则写空字符串或不写）。
- 老书累加库存（`increment_stock`）：不修改 `introduction`。
- 老书跳过（`skip`）：不修改 `introduction`。
- 老书覆盖字段（`overwrite`）：当 Excel 该列有值时，更新 `introduction`。

## 4. 页面/前端改动点（vue-vben-admin）
### 4.1 新增/编辑抽屉表单
- 新增表单项：`简介`
  - 组件：`Input`（`type: 'textarea'`）
  - `maxlength: 300`、`showWordLimit: true`、`autosize`（3~6 行）
- 提交时把 `introduction` 一并提交到后端。

### 4.2 列表表格
- 不新增简介列（保持表格紧凑）。

### 4.3 导入预览
- 不在导入预览表格中展示简介列（避免列过宽）。
- 仅保证导入时能写入/更新简介字段。

## 5. 数据内容来源说明（论文口径建议）
- 系统不自动联网抓取简介，简介由管理员手动录入或通过 Excel 导入维护。
- 如需演示数据，可在导入 Excel 中填写（内容可参考公开资料，但建议自行改写，避免直接复制原文带来的版权风险）。

## 6. 验收点
- 新增图书：在抽屉中可输入简介，保存后再次编辑能正确回填；简介超过 300 字会提示错误并保存失败。
- 编辑图书：可修改简介并保存生效。
- 列表表格：不出现简介列。
- 批量导入：
  - Excel 带 `introduction/简介` 列时，新书能写入简介；
  - 老书在 `overwrite` 策略下可更新简介，在 `increment_stock/skip` 下简介不变。

## 7. 冒烟记录（开发完成后补充）

# v1.1（2026-03-07）简介数据回填（优化）

## 1. 目标
- 基于当前 MongoDB `books` 数据，为“简介为空”的图书批量补齐 `introduction`。
- 明显测试数据（标题/作者含测试关键词等）保持简介为空。
- 不覆盖已有简介（避免把你手动编辑/导入的内容冲掉）。

## 2. 数据来源与规则
- 数据来源：优先从 Open Library（HTTP 接口）按 ISBN 获取公开的书籍描述/信息；缺失时使用本地模板生成简短简介。
- 长度：最终写入强制截断为 `<= 300` 字（与接口校验一致）。
- 跳过规则（视为测试数据，不回填）：
  - `title/author/isbn` 中包含：`测试`、`test`、`demo`、`mock`、`前导0`、`占位` 等（不区分大小写）

## 3. 实现改动点（backend）
- 新增脚本：`backend/src/scripts/backfill-book-introductions.ts`
  - 支持 `--dry-run` 预览、`--limit` 限制条数（防止误操作）。
  - 默认行为：只更新 `introduction` 为空/不存在的记录，且非测试数据。
- `backend/package.json` 新增脚本命令（便于执行）。

## 4. 验收点
- 执行脚本后：
  - 非测试数据的图书简介基本补齐（允许少量因外部资料缺失而仍为空）。
  - 测试数据（如标题含“测试/前导0”等）简介仍为空。
  - 已有简介的记录内容不被覆盖。
