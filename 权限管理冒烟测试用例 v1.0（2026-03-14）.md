# 权限管理冒烟测试用例 v1.0（2026-03-14）

目标：覆盖“入库 → 用户预约（待取书）→ 管理员确认借出 → 用户/管理员侧记录一致 → 还书 → 库存/状态更新”的主链路，并验证硬权限生效。

## 0. 前置条件
- MongoDB 可连接（默认 `mongodb://127.0.0.1:27017`，库名 `library`）。
- 后端已启动（默认 `http://localhost:3000`）。
- 前端可选启动，用于 UI 冒烟（`http://localhost:<vite-port>`）。

建议准备账号：
- 管理员：`admin/123456`（后端启动会初始化）
- 普通用户：本用例会通过管理员接口创建 `reader1/123456`

## 1. 变量约定
- `BASE_URL`：后端地址，例如 `http://localhost:3000/api`
- `ADMIN_TOKEN`：管理员登录拿到的 `accessToken`
- `USER_TOKEN`：普通用户登录拿到的 `accessToken`
- `ISBN`：本次入库的 ISBN（建议用不冲突的 13 位数字）
- `RECORD_ID`：预约/借阅记录的 `record_id`

## 2. API 冒烟（推荐，最可复现）
说明：返回结构是 `{ code, data, error, message }`，需要从 `data` 中取值。没有 jq 的情况下建议直接手动复制 token/record_id。

### 2.1 管理员登录获取 token
`POST ${BASE_URL}/auth/login`

Body：
```json
{ "username": "admin", "password": "123456" }
```

期望：
- HTTP 200，`data.accessToken` 不为空，保存为 `ADMIN_TOKEN`。

### 2.2 管理员创建普通用户（reader1）
`POST ${BASE_URL}/users`（管理员接口）

Body（手机号需唯一）：
```json
{
  "username": "reader1",
  "phone": "13900000001",
  "role": "user",
  "status": 1,
  "credit_score": 100,
  "avatar": ""
}
```

Header：
- `Authorization: Bearer ${ADMIN_TOKEN}`

期望：
- HTTP 200，返回用户信息（后端会给默认密码 `123456`）。

备注：
- 若返回 `409 手机号已存在/username 已存在`，请换 `username/phone` 再试。

### 2.3 管理员入库新书（库存=1）
`POST ${BASE_URL}/books`（管理员接口）

Body（`ISBN` 自填）：
```json
{
  "isbn": "9780000000001",
  "title": "冒烟测试用书",
  "author": "TestAuthor",
  "category": "其他",
  "cover_url": "",
  "total_stock": 1,
  "current_stock": 1,
  "introduction": ""
}
```

Header：
- `Authorization: Bearer ${ADMIN_TOKEN}`

期望：
- HTTP 200，返回 `current_stock=1`。

### 2.4 普通用户登录获取 token
`POST ${BASE_URL}/auth/login`

Body：
```json
{ "username": "reader1", "password": "123456" }
```

期望：
- HTTP 200，保存 `USER_TOKEN`。

### 2.5 用户预约图书（待取书 reserved）
`POST ${BASE_URL}/borrows/reserve`

Body：
```json
{ "isbn": "9780000000001" }
```

Header：
- `Authorization: Bearer ${USER_TOKEN}`

期望：
- HTTP 200
- 返回 `data.record.status = "reserved"`，并包含 `record_id`（保存为 `RECORD_ID`）
- 返回 `data.book.current_stock` 变为 `0`

### 2.6 用户查看“我的借阅/预约记录”
`GET ${BASE_URL}/borrows/my?page=1&pageSize=20`

Header：
- `Authorization: Bearer ${USER_TOKEN}`

期望：
- HTTP 200
- 列表里存在 `record_id=${RECORD_ID}` 的记录
- `status` 为 `reserved`（待取书）

### 2.7 管理员确认借出（把 reserved 转成 borrowed，不重复扣库存）
说明：当前实现沿用管理员借书接口 `POST /borrows/borrow`；若命中同一用户同一本书的 `reserved` 记录，会直接转为 `borrowed`。

`POST ${BASE_URL}/borrows/borrow`（管理员接口）

Body：
```json
{ "isbn": "9780000000001", "username": "reader1" }
```

Header：
- `Authorization: Bearer ${ADMIN_TOKEN}`

期望：
- HTTP 200
- 返回 `data.record.status = "borrowed"`
- 库存仍为 `0`（不会再次扣减）

### 2.8 管理员侧借阅管理列表能看到记录
`GET ${BASE_URL}/borrows?page=1&pageSize=20&username=reader1`

Header：
- `Authorization: Bearer ${ADMIN_TOKEN}`

期望：
- HTTP 200
- `items` 中存在 `record_id=${RECORD_ID}`（同一条记录，状态为 borrowed/overdue 等）

### 2.9 管理员还书（库存回补）
`PUT ${BASE_URL}/borrows/${RECORD_ID}/return`（管理员接口）

Body：
```json
{ "fine_amount": 0 }
```

Header：
- `Authorization: Bearer ${ADMIN_TOKEN}`

期望：
- HTTP 200
- `data` 为“记录对象”（字段形态与 `GET /api/borrows` 返回的 items 单项一致）
  - `status = "returned"` 且 `return_date` 存在
- 说明：当前接口响应不包含图书库存字段；如需验证库存回补，追加执行一次（管理员 token）：
  - `GET ${BASE_URL}/books?page=1&pageSize=20&isbn=${ISBN}`，期望第一条 `current_stock = 1`

### 2.10 用户侧借阅记录同步更新
`GET ${BASE_URL}/borrows/my?page=1&pageSize=20`

Header：
- `Authorization: Bearer ${USER_TOKEN}`

期望：
- HTTP 200
- `record_id=${RECORD_ID}` 的记录 `status = "returned"`

## 3. 硬权限校验用例（必须）
### 3.1 普通用户访问管理员接口必须 403
用 `USER_TOKEN` 调用以下任意一个：
- `GET ${BASE_URL}/users?page=1&pageSize=20`
- `GET ${BASE_URL}/analytics/overview`
- `POST ${BASE_URL}/books`（入库）
- `GET ${BASE_URL}/borrows`（管理端列表）

期望：
- HTTP 403（Forbidden）

## 4. UI 冒烟（可选）
目的：确认菜单隔离 + 两端页面链路可用。

### 4.1 管理员侧
- 登录管理员账号
- 新书入库（库存=1）
- 借阅管理：为 `reader1` 借书（若已预约则会确认借出）
- 借阅管理：还书（库存回补）

### 4.2 普通用户侧
- 登录普通用户账号
- 图书预定：预约该书（预约后借阅记录出现待取书）
- 借阅记录：确认状态变化（reserved → borrowed → returned）

## 5. 清理数据（可选）
避免污染演示库数据。
- 书籍下架：`PUT ${BASE_URL}/books/${ISBN}/shelf`（管理员 token）
  - Body：`{ "is_deleted": true }`
- 删除用户：`DELETE ${BASE_URL}/users/:id`（管理员 token）
  - `:id` 为创建用户接口返回的 `_id`
