## 2026-03-15 v1.0

### 范围
- 用户管理列表新增“排序方式”切换
- 默认保持“最近注册”
- 支持“按角色分组”查看用户

### 页面改动点
- 搜索区新增 `sortMode`
- 选项：
  - `recent`：最近注册
  - `role`：按角色分组
- 默认值：`recent`

### 接口改动点
- `GET /api/users` 新增可选查询参数 `sortMode?: 'recent' | 'role'`
- `recent`：按 `created_at desc`
- `role`：按 `super -> admin -> user` 分组，同组内按 `created_at desc`

### 实现约束
- 不改变现有分页、角色筛选、状态筛选、注册时间筛选逻辑
- Redis 列表缓存 key 需要纳入 `sortMode`
- 不新增新的表格列排序交互，仅提供搜索区统一切换

### 验收点
- 不传 `sortMode` 时，列表行为与当前一致
- 选择“按角色分组”后，列表顺序为 `super -> admin -> user`
- 同一角色内，注册时间新的排前面
- 翻页、筛选、刷新后排序方式保持生效

## 2026-03-15 v1.1

### 调整原因
- 用户管理排序更适合放在表格表头，而不是搜索表单
- 搜索区的“重置”与工具栏“刷新”只应处理筛选与数据刷新，不应把排序误当成表单条件

### 范围
- 删除搜索区“排序方式”
- 改为通过表头对“角色”“注册时间”执行远程排序

### 页面改动点
- 搜索区移除 `sortMode`
- 表头开启远程排序：
  - `角色`
  - `注册时间`
- 默认排序：
  - `注册时间 desc`

### 接口改动点
- `GET /api/users` 调整为：
  - `sortBy?: 'created_at' | 'role'`
  - `sortOrder?: 'asc' | 'desc'`
- 排序规则：
  - `created_at`
    - `asc`：注册时间从旧到新
    - `desc`：注册时间从新到旧
  - `role`
    - `asc`：`super -> admin -> user`
    - `desc`：`user -> admin -> super`
- `role` 排序下，同角色内固定按 `created_at desc`

### 实现约束
- “重置”仅重置搜索筛选项，不依赖表单控制排序
- “刷新”保留当前表头排序状态
- 当表头排序被清空时，回退为默认 `created_at desc`

### 验收点
- 页面初次进入默认按注册时间倒序
- 点击“注册时间”列头可切换升序/降序
- 点击“角色”列头可按业务角色优先级升序/降序
- 点击“重置”后，筛选项重置但当前排序能力仍正常
- 点击“刷新”后，当前排序状态保持有效

## 2026-03-15 v1.2

### 调整原因
- 用户在表头完成排序后，点击“重置”或工具栏“刷新”会误回退到默认排序
- 原因是查询逻辑把“本次请求未携带 sort 参数”和“用户主动清空排序”混为一类处理

### 改动点
- 用户管理列表查询逻辑调整为：
  - 本次请求未携带 `sort` 参数时，沿用当前排序状态
  - 仅当排序事件明确指向 `role/created_at` 且顺序被清空时，回退到默认 `created_at desc`

### 验收点
- 点击表头完成排序后，“重置”不清空当前排序
- 点击工具栏“刷新”不清空当前排序
- 用户主动清空表头排序时，列表回退到默认 `created_at desc`

## 2026-03-15 v1.4

### 调整原因
- 手动回显表头排序状态后，用户无法通过第三次点击清空排序
- 原因是业务逻辑把“用户主动清空排序”立即回写成默认排序，打断了表格组件原生三态排序

### 改动点
- 排序状态调整为 `active + field + order`
- 用户主动清空排序时：
  - 保留默认请求排序用于稳定列表顺序
  - 但不再强制点亮表头默认排序状态
- `重置`、`刷新` 在有排序时继续保留当前激活状态；在已清空排序时保持无激活状态

### 验收点
- 表头支持 `asc -> desc -> 无排序` 三态切换
- 清空排序后，表头不再被立即强制回显为默认排序
- `重置`、`刷新` 仍不破坏当前排序或清空状态

## 2026-03-15 v1.5

### 调整原因
- 实际交互中，“无排序表头 + 默认顺序数据”容易造成理解混乱
- 为降低心智负担，用户管理改回更简单、可预期的默认行为

### 改动点
- 删除额外的排序状态保留逻辑
- 以下操作统一恢复默认排序并回显默认表头激活态：
  - 点击表头第三次清空排序
  - 点击“重置”
  - 点击工具栏“刷新”
- 默认排序：
  - `注册时间 desc`

### 验收点
- 表头点击升序/降序正常
- 第三次点击清空排序后，恢复默认排序 `注册时间 desc`
- 点击“重置”后，筛选与排序都恢复默认
- 点击工具栏“刷新”后，排序恢复默认

## 2026-03-15 v1.3

### 调整原因
- 用户完成排序后，`重置/刷新` 虽然已不改变实际查询顺序，但表头升降序图标的激活状态会丢失
- 这是表格组件的展示状态未自动同步回当前业务排序状态

### 改动点
- 用户管理列表在每次查询成功后，手动将当前 `gridSortState` 回写到表格表头
- 使用表格实例 `sort()` 同步 UI 激活状态，不额外触发新的远程查询

### 验收点
- 点击“角色/注册时间”排序后，表头激活状态正确显示
- 点击“重置”后，当前排序的表头激活状态仍保留
- 点击工具栏“刷新”后，当前排序的表头激活状态仍保留
- 用户主动清空排序后，表头回到默认 `注册时间 desc` 的激活状态
